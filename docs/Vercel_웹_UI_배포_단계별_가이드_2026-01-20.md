1. **빌드 로그 확인**: Vercel 대시보드 → Deployments → 실패한 배포 클릭 → Logs 확인
2. **환경 변수 확인**: 환경 변수가 올바르게 설정되었는지 확인
3. **로컬 빌드 테스트**: `npm run build` 로컬에서 실행하여 에러 확인

### 환경 변수 미적용 시

1. **환경 변수 재설정**: Vercel 대시보드 → Settings → Environment Variables
2. **재배포**: "Redeploy" 버튼 클릭

1. **VPS 상태 확인**: `http://148.230.97.246:11434/api/tags` 접속 확인
